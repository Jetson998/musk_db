# Releases

Release-specific patches, configs, scripts, and deployment records for musk_db.

Each release folder `YYYY-MM-DD-short-name/` must include a `README.md` with:
reason, affected files/services, implementation summary, verification plan,
rollback plan, deployment result, and backup paths/artifacts.

Template: `../docs/templates/minimal-release-record.md` (workflow Section 20).
